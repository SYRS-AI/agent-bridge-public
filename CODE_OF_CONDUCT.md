# Code Of Conduct

This project expects respectful, constructive collaboration.

## Our Standards

Examples of behavior that help the project:

- assume good intent
- give direct technical feedback without hostility
- document reasoning clearly
- focus disagreement on design and implementation, not people
- help keep review and issue threads actionable

Examples of unacceptable behavior:

- harassment, insults, or personal attacks
- deliberate intimidation
- bad-faith derailment
- publishing private information without permission
- repetitive noise that blocks collaboration

## Enforcement

Project maintainers may edit, hide, lock, or remove comments, issues, pull requests, and other contributions that violate these standards.

## Scope

This code of conduct applies to project spaces, including issues, pull requests, discussions, and review threads.

## Reporting

If you encounter a serious conduct problem, contact the maintainers through a private channel associated with the repository host instead of escalating in a public thread.
